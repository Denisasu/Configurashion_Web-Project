Артефакт: User-Experience Storyboard
This work product describes how a user interacts with the system in terms of User-Experience Elements (e.g., screens, input forms) to perform system functionality.
Типы рабочих продуктов: Элементы модели
Назначение

A User-Experience Storyboard demonstrates how a Use Case is realized in terms of User-Experience Elements. User-Experience Storyboards can be viewed as a more detailed look at the user-interface boundary Analysis Classes that appear in the analysis-level Use-Case Realizations. If Storyboards were also created for those Use Cases, then the User-Experience Storyboards can be considered realizations of those Storyboards, expressed in terms of the User-Experience Elements.

Collectively, the User-Experience Storyboards describe the dynamics of the User-Experience Model.

Описание
Основное описание

A User-Experience Storyboard should be defined for each Use Case with a user-experience aspect. Defining a User-Experience Storyboard for each Use Case directly supports a use-case-driven approach for software engineering in that the design of the user interface is driven by the Use Cases that the system is required to provide. Defining a User-Experience Storyboard for each Use Case guarantees that the user-interface design will support the interactions required in the Use Cases.

The following people use the User-Experience Storyboard:

  • User-Interface Designers, to design the user interface and to build a prototype of the user interface.
  • Designers of the classes that provide functionality for the user interface. They use this information to understand the roles and interactions of the user interface elements, so they can properly design the classes that will implement the user interface.
  • Those who design the next version of the system to understand how the system carries out the flow of events in terms of user interface elements.
  • Those who test the system's features.
  • The manager to plan and follow up the analysis and design work.
Краткая схема

A User-Experience Storyboard contains:

  • Flows of Events: A textual description of how the user interacts with User-Experience Elements to perform the system functionality. The Flows of Events can be used as the "scripts" for the User-Experience Storyboard flow diagrams. These descriptions may be augmented with usability aspects to clarify and outline the allocation of usability requirements onto the user-experience elements.
  • Flow Diagrams:The interaction diagrams (sequence and/or collaboration diagrams) describing the storyboard in terms of collaborating User-Experience Elements.
  • Participants Diagrams: The diagrams describing the navigation paths between the User-Experience Elements participating in the storyboard. These diagrams can be considered "mini" User-Experience Navigation Maps, where the scope of the diagram is smaller than the entire application. 
  • References to Usability Requirements: References all applicable usability requirements that need to be taken care of during user-interface prototyping and implementation. Examples are maximum execution time (e.g., how long it should take a trained user to execute a scenario), and maximum error rate (e.g. how many errors a trained user is allowed to make when executing a scenario).
  • (optional) Reference to a User-Interface Prototype: A User-Experience Storyboard may refer to a User-Interface Prototype (or the actual user interface) that exemplifies the storyboard, once one is available.

The contents of the User-Experience Storyboards should be tailored to support project needs.  For example, the Flow Description can be documented as the "script" that is included as part of the storyboard's Flow Diagrams.

For guidelines on developing the User-Experience Storyboards, see Guideline: User-Experience Storyboard.
