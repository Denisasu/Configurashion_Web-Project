Артефакт: User-Experience Element
This work product is an abstraction that represents a user interface instance that a user interacts with.
Типы рабочих продуктов: Элементы модели
Назначение

User-Experience Elements are the major modeling elements of the User-Experience Model.  The general term User-Experience Element is used for the artifact name, and stereotypes are used to indicate specific types of user-experience elements.

There are two main types of User-Experience Elements:

  • Screen: Represents everything the application generates to the user in a single client window, (including a header, footer, sub-windows, etc.). It represents a consistent and coherent user-interface element.
    In a Web application, a screen could represent a frameset with multiple HTML page resources rendered inside.
  • Input form: Represents a part of a screen where the user enters information. Input forms are always part of a screen.

Screens are the main type of User-Experience Elements, since all User-Experience Elements are either screens or they are contained in screens.

Decide which User-Experience Element properties to model/use and how to model them.  Decide on what level of formality to use for the modeling of the static User-Experience Element operations (the navigate to and display messages).  Since these operations do not really represent user actions, the formality with which there are represented can be minimized. 

For information on developing User-Experience Elements, see Guideline: User-Experience Element.
