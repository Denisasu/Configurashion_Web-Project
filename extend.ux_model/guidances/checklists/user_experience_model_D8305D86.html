Справочная таблица: User-Experience Model
This checklist contains a list of things to check when reviewing the User-Experience Model.

Элементы справочной таблицы
General
  • The objectives of the model are clearly stated and visible.
  • The model is at an appropriate level of detail given the model objectives.
    The level of detail is consistent with the project-specific guidelines.
  • The model's use of modeling constructs is appropriate to the problem at hand.
    The use of modeling constructs is consistent with the project-specific guidelines.
  • The model is as simple as possible while still achieving the goals of the model.
  • The model appears to be able to accommodate reasonably expected future change.
  • The design is appropriate to the task at hand (neither too complex nor too advanced).
    The level of design is consistent with the project-specific guidelines.
  • The overall structure of the User-Experience Model is well-formed and appears to be understandable and maintainable
  • The User-Experience Model supports the functional and usability requirements made on the system.
  • The User-Experience Elements and User-Experience Storyboard flows of events are consistent with the project-specific guidelines.
  • The contents of any packages should be examined to ensure high cohesion within packaging elements.

Internal Consistency Rules
  • Every defined User-Experience Element must be represented on at least one User-Experience Model Navigation Map diagram.
  • Every defined User-Experience Element must be represented on at least on User-Experience Storyboard flow diagram.

  • The User-Experience Navigation Map and the User-Experience Storyboards must be consistent.  The consistency rules between them are the same as the consistency rules between the User-Experience Storyboard flow diagrams and the User-Experience Storyboard participant diagrams.  This is because the User-Experience Navigation Map can be considered a summary (or "global view") of all User-Experience Storyboard participant diagrams. For more information, see the "Diagram Consistency Rules" section of the Checklist: User-Experience Storyboard.
