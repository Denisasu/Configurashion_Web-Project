Отчет: User-Experience Storyboard
This report is used by various people interested in the User-Experience Storyboard, such as User-Interface Designers, Designers, Testers, and Project Managers.

Связанные элементы
  • User-Experience Storyboard
Основное описание

1. Brief Description
 A brief description of the system functionality whose user interactions are described in the User-Experience Storyboard. If the User-Experience Storyboard realizes a specific Use Case, then that Use Case should be referenced.

2. Flows of Events (optional)
 The User-Experience Storyboard's flows of events. If the flows of events are captured as documentation on the Flow Diagrams, then this section can be eliminated.

3. Flow Diagrams
 The User-Experience Storyboard's flow diagrams. One or more interaction diagrams (sequence and/or collaboration diagrams) describing the User-Experience Storyboard in terms of collaborating User-Experience Elements 

4. Participants Diagram
 The class diagrams describing the navigation paths between the User-Experience Elements participating in the User-Experience Storyboard.

5. Participating User-Experience Elements
 User-Experience Elements (classes) that participate in interaction diagrams of the User-Experience Storyboard. For each User-Experience Element, include its class name and brief description.

6. Usability Requirements
 References to usability requirements that affect the User-Experience Storyboard.

7. References to the User-Interface Prototype (optional)
 If a User-Interface Prototype exists that further clarifies the User-Experience Storyboard, it should be referenced here.
