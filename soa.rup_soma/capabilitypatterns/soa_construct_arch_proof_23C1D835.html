Дескриптор задачи: Construct Architectural Proof-of-Concept (SOA)
This task defines how to develop an architectural proof-of-concept, for a SOA solution, based on existing architectural requirements and risk profile.

Шаги
Выбрать подход к построению

Выберите техники, которые будут использоваться для построения Архитектурного опытного образца, например:

  • Концептуальное моделирование
  • 'Быстрое' создание прототипа
  • Имитация
  • Автоматическая трансляция спецификаций в исходный код
  • 'Исполняемые' спецификации
  • Построение   'гвоздей' как прототипов - вертикальных срезов через все уровни

Разработчик программного обеспечения должен понимать эти модели, чтобы обнаружить какие-нибудь проблемы и найти решения.

Выбрать ресурсы и технологии для архитектурного опытного образца

Разработчик программного обеспечения должен выбрать из определенных в "Задаче: Архитектурный анализ" ресурсов и технологий те, которые будут использоваться при построении Архитектурного опытного образца.

Построение доказательства концепции архитектуры

С помощью выбранных для построения технологий и ресурсов разработчик программного обеспечения компонует Архитектурный опытный образец, который удовлетворяет существенным для архитектуры требованиям, зафиксированным в стандартных реализациях вариантов использования, обзоре моделей проекта и развертывания, и документации по архитектуре программного обеспечения.
