Концепция: Представление реализации
Представление реализации - это архитектурное представление, охватывающее все подсистемы модели реализации и представляющее собой диаграмму компонентов, иллюстрирующую взаимосвязь подсистем, зависимости между ними и их иерархическую структуру.

Основное описание

Представление реализации - это одно из пяти архитектурных представлений системы. Помимо этого, выделяют логическое представление, представление вариантов использования, представление процессов и представление развертывания.

Представление реализации предназначено для отслеживания архитектурных решений, принятых для реализации. Как правило, в представление реализации входят следующие компоненты:

  • список всех подсистем в модели реализации
  • диаграммы компонентов, иллюстрирующие принципы распределения подсистем по уровням и иерархическим структурам.
  • иллюстрации зависимостей импорта между подсистемами

Представление реализации применяется для выполнения следующих задач:

  • распределение работы по реализации между отдельными сотрудниками, группами и подрядчиками
  • оценка объема кода, подлежащего разработке, изменению или удалению
  • обоснование широкомасштабного многократного использования ресурсов
  • сравнительный анализ стратегий выпуска

Представление реализации и другие архитектурные представления описаны в документе архитектуры программного обеспечения.
