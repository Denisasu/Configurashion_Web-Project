Настройка среды тестирования

В приложении J2EE для полнофункционального тестирования отдельного компонента может понадобиться собрать компонент в Приложение J2EE и фактически развернуть его. Конечно, в этой ситуации выполняется минимальная сборка и развертывание, необходимые для выполнения компонента в среде тестирования. Дополнительная информация о сборке приложений J2EE приведена в разделе Наставник по инструменту: Сборка модулей и приложений J2EE с помощью Rational XDE Developer - Java Platform Edition. Дополнительная информация о развертывании приложений J2EE приведена в разделе Памятка по инструменту: Развертывание модулей и приложений J2EE с помощью Rational XDE Developer - Java Platform Edition.

Если в системе установлен WebSphere Studio Application Developer, можно тестировать компоненты во встроенной среде WebSphere Test Environment (WTE). Выберите компоненты в проекции J2EE браузера J2EE и вызовите команду "Выполнить на сервере" контекстного меню. За дополнительной информацией обратитесь к Web-справке о работе компонента WebSphere Studio Application Developer.

Для отладки приложения на отдельном сервере приложений, запустите этот сервер с подключенной функцией отладки. За инструкциями обратитесь к документации сервера. После развертывания приложения подключите к серверу отладчик Eclipse и выполните отладку приложения. Заметьте, что большинство серверов приложений не предоставляют возможности отладки страниц JSP (которые скомпилированы в код Java). За подробностями обратитесь к документации сервера.

Выполнение теста

После развертывания приложения выполните отладку компонентов в обычном режиме с помощью встроенного отладчика Java и других доступных инструментов по отладке и анализу.

Заметьте, что при использовании WebSphere Studio Application Developer and XDE можно применять функцию Trace Rational Purify and XDE для отладки и трассировки выполняющихся компонентов J2EE в среде тестирования WebSphere Test Environment (WTE), которая является частью Application Developer. Заметьте также, что XDE Trace не поддерживает компонентов, выполняющихся на удаленных серверах приложений J2EE. Более подробная информация находится в разделе Анализ поведения во время выполнения с помощью Rational PurifyPlus. В настоящий момент невозможно использовать Trace или PurifyPlus для страниц JSP.
